# Бюро находок учебного заведения

Веб‑приложение на **Python + Flask + SQLite**: учёт найденных вещей и статусов выдачи (Найдено / Забрано / Передано).

## Требования к отчёту

- Данные с экрана — **карточки**, не таблица; **ID не показываются** (хранятся в `data-id` у карточки).
- **CRUD** по БД: администратор добавляет/правит/удаляет находки и места, меняет статус (запись в `status_log`).
- **Вход по логину**: роль **администратор** (полный доступ) и **пользователь** (только просмотр карточек).

## Таблицы SQLite

| Таблица       | Назначение |
|---------------|------------|
| `locations`   | кабинет, этаж, корпус |
| `found_items` | название, описание, дата находки, ссылка на место |
| `status_log`  | история статусов по находке |
| `users`       | логин, хеш пароля, роль |

БД создаётся при первом запуске: `instance/lost_found.sqlite3`.

## Запуск (Windows, PowerShell)

```powershell
cd путь\к\upafisha
py -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
python app.py
```

Откройте в браузере: http://127.0.0.1:5000

## Учётные записи

- Регистрация через сайт создаёт только роль **пользователь** (просмотр).
- При первом запуске создаётся администратор: логин **`admin`**, пароль **`admin`** (смените в продакшене и задайте `SECRET_KEY` в переменных окружения).

## Маршруты

- `/login`, `/register`, `/logout`
- `/items` — список карточек (нужен вход)
- `/admin/locations` — места (CRUD)
- `/admin/items/new`, правка, удаление, смена статуса — только админ
